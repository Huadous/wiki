# R8

## 基本信息
- Type: product
- Source: [[sources/java-lite|java-lite]]

## 描述
R8 是 Android 平台默认的代码缩减（shrinking）和混淆（obfuscation）工具，作为 ProGuard 的替代品内置于 Android Gradle Plugin 之中，主要用于在生产构建中优化 APK 体积并提升代码安全性。默认情况下，R8 会对类、方法和字段名进行重命名混淆（minify），这在 [[entities/protocolbuffersprotobuf|Protocol Buffers]] 生成的代码场景下会引发兼容性问题：[[sources/java-lite|java-lite]] 在内部使用反射机制来避免为每个消息类型生成 `hashCode`、`equals` 与序列化/反序列化方法，一旦 R8 混淆了生成代码的字段名，反射查找便会失败。该问题在 [[entities/protocolbuffersprotobuf|protobuf]] GitHub 项目（#6463）以及 R8 issue tracker（issuetracker.google.com/144631039）中均有相关跟踪，运行时通常抛出 `java.lang.RuntimeException: Field {NAME}_ for {CLASS} not found. Known fields are [ {FIELDS} ]`（位于 `MessageSchema.java`）。
