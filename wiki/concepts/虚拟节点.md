# 虚拟节点

## 定义
虚拟节点（Virtual Node）是一致性哈希中用于解决节点数量较少时区间划分不均衡和单点故障压力过大问题而引入的技术。其做法是为每个真实 server 计算 m 个不同的 hash 值，从而将 32 位整数值域划分为 n*m 个更细的区间。当 key 落到某个区间时，分流到对应的 server 上，这些额外的 hash 值使得区间划分更加均匀。

## 关键特征
- 为每个真实 server 计算 m 个不同的 hash 值，将 32 位值域划分为 n*m 个更细的区间
- 节点下线时：被移除 server 的 m 个虚拟区间会分别合入相邻区间，请求较为平均地分散到其他 server
- 节点上线时：分割 m 个现有区间并从各 server 均匀转移部分请求
- 有效缓解节点数量较少时区间划分不均和单点故障压力过大的问题
- brpc 默认每个 server 配置 100 个虚拟节点

## 应用
- brpc 一致性哈希负载均衡：为每个真实 server 创建多个虚拟节点，使请求在集群中更均匀地分布
- 通过 `-chash_num_replicas` 参数可设置默认的虚拟节点个数，默认值为 100
- 特殊场景下，可在 `load_balancer_name` 中追加 `replicas=<num>` 子参数自定义虚拟节点数量
- 缓解一致性哈希在节点数量较少时扩容/缩容引发的负载倾斜问题
