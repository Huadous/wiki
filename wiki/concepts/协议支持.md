## Description
brpc 的协议支持覆盖客户端与服务端两个维度。在客户端侧，[[entities/brpc::Channel|brpc::Channel]] 通过 `ChannelOptions.protocol` 字段配置通信协议，该字段既接受 enum 类型（如 `PROTOCOL_BAIDU_STD`、`PROTOCOL_H2`）也接受字符串（如 `"baidu_std"`、`"h2"`），默认协议为 `baidu_std`。用户切换协议时无需修改业务代码，仅需调整该配置即可，这为协议迁移、灰度发布以及多协议并行测试提供了极大便利。每种协议在 brpc 中都有其默认的连接方式，例如 `baidu_std` 与 `h2`（gRPC）默认为单连接，而 `http` 则使用短连接，连接池类协议则采用连接池方式。

在服务端侧，brpc server 会自动尝试其支持的所有协议而无需用户通过配置显式指定，客户端可通过 `cntl->protocol()` 获取当前连接实际使用的协议类型。一个 listen 端口可以同时承载不同协议的连接（单端口多协议），无需为每种协议单独分配端口，这大大简化了部署与运维。默认启用的协议包括 baidu_std、streaming_rpc、http/1.0、http/1.1、h2c、h2（gRPC）、rtmp、hulu_pbrpc、sofa_pbrpc 等；nova_pbrpc、public_pbrpc、nshead_mcpack 等则需通过 `ServerOptions.nshead_service` 显式开启。brpc 同时支持 ubrpc（compack/mcpack2）、nshead 系列、esp、thrift、memcache、redis 等多种协议，使其能在 RPC、HTTP 服务、缓存访问、消息队列等多种场景中充当统一通信层。
