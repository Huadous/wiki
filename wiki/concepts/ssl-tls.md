## Description
SSL 作为通用安全传输层作用于 brpc 的协议层之下，因此一旦开启，所有上层协议（包括 HTTP/HTTP/2、gRPC、hulu-pbrpc 等）均可获得加密传输能力，而非仅限于 HTTP。服务端启用 SSL 需要配置 `ServerOptions.ssl_options` 并提供 `default_cert`；当需要基于请求域名（SNI）动态选择证书时，可将多张证书加载到 `certs` 中，未匹配到任何证书的请求可指定由 `default_cert` 处理，否则将被丢弃。同一个端口在开启 SSL 后仍可接受非 SSL 访问。客户端启用 SSL 的通用方法在 `client.md` 中详述，为方便使用，brpc 对以 https 开头的 URI 会自动开启 SSL，无需额外配置。当客户端访问启用身份验证的 server 时，需根据 server 的认证方式生成 `auth_data`，并将其设置为 HTTP Header "Authorization" 的值（对应 curl 的 `-H "Authorization : <auth_data>"` 选项）。
