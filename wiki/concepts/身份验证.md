## Description
身份验证在 brpc 中是一个双向协作的机制：Server 端按照连接粒度对客户端进行身份校验，而客户端则负责生成对应的凭据。Server 端在收到新连接上的第一个请求时，会解析请求中的身份信息并调用用户实现的 `Authenticator::VerifyCredential` 方法进行验证；验证通过后，`AuthContext` 会被保存到该连接上，可通过 `controller->auth_context()` 获取，后续同一连接上的请求默认直接通过验证而不再产生认证开销。客户端侧的认证方式分为两类：**基于请求的认证**（每条请求都携带认证信息）和**基于连接的认证**（连接建立后发送一次认证包，后续请求复用该凭据），其中基于连接的方式在 single connection 或 connection pool 场景下性能更优。brpc 为客户端提供了 `Authenticator` 接口，用户需实现 `virtual int GenerateCredential(std::string* auth_str) const = 0;` 来生成凭据，并将实现类通过 `ChannelOptions` 注入。目前 brpc 中只有 `baidu_std`（默认协议）、`HTTP`、`hulu_pbrpc`、`ESP` 等少数协议支持内置的客户端认证。在多租户或分级权限场景中，身份验证能够精确识别发起连接的客户端身份，并可与传输层安全机制（如 [[concepts/ssl|SSL]]）结合构建多层防御体系。
