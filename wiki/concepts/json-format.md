JSON representation 是 Protocol Buffers 的一种文本序列化形式，使得 protobuf 消息能够以人类可读的 JSON 格式进行传输和存储。在 proto3 中，消息在二进制、JSON 和 TextFormat 三种表示之间没有差异，可以自由转换。JSON 格式的字段与 protobuf 字段通过匹配字段名和消息结构对应，但 JSON 中未在 protobuf 中定义的字段会被丢弃。这种格式在 HTTP/h2 场景下被广泛使用，允许不同语言的服务通过 JSON 进行交互，如通过 curl 工具直接发送 JSON 请求访问 protobuf 服务。使用保留字段名时需要注意对 JSON 解析的影响，建议同时保留字段名以确保 JSON 和 TextFormat 编码的兼容性。
