## Description
JSON 是 Protocol Buffers 支持的一种人类可读的命名字段映射格式，文件扩展名为 `.json`。在 brpc 中，使用 protobuf 的服务通常可以通过 HTTP/h2 + JSON 方式访问，存储在 body 中的 JSON 字符串可以与对应的 protobuf 消息相互转换。JSON 字段通过匹配的名字和消息结构与 pb 字段一一对应，转换时必须包含 pb 中的 required 字段，否则转换会失败且请求会被拒绝；同时，JSON 中可以包含 pb 中未定义的字段，但这些字段会被丢弃而不会作为 unknown fields 存储到 pb 中。

JSON 的转换行为受运行时选项影响，例如开启 `-pb_enum_as_number` 后，pb 中的 enum 会转化为它的数值而不是名字，该选项同时影响客户端发出的请求和服务器返回的响应。在性能方面，可以通过设置 `Content-Type: application/proto` 使用 protobuf 序列化数据以获得更高的序列化效率。

相对于 tag-value stream（wire format）和 TextFormat，JSON 是一种更严格的格式，无法有效表达 wire format 或 TextFormat 中的一些语义。JSON 定义了 null 值，可以用来表示"已定义但未设置"的字段，这与 no presence 规则下其他格式的行为不同；此外，JSON 元素在语义上是无序的，每个成员必须有唯一的名称，因此 repeated 字段值在 JSON 中不合法，这与 TextFormat 对 repeated 字段的规则不同。JSON 也无法明确地表达 oneof 字段的"最后一个胜出"规则，当 oneof 多个 case 同时存在时会出现歧义。
