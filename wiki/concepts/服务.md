# 服务

## 定义
在 baidu_std 协议中，服务（Service）是指通过某个 IP 地址上的某个端口发布的 RPC 服务单元。服务以名字标识，是 RPC 方法的逻辑分组单位。

## 关键特征
- 所有 RPC 服务都在某个 IP 地址上通过某个端口发布
- 一个端口可以同时发布多个服务
- 服务名采用 UpperCamelCase 命名风格，由大小写字母和数字组成
- 服务名长度不超过 64 个字符
- 每个服务可包含一个或多个方法（方法命名不做强制格式规定）
- 由"服务名 + 方法名 + IP + 端口"构成的四元组唯一地标识了一个 RPC 方法

## 应用
- 在 baidu_std 协议下，服务器端通过单一端口发布多个逻辑服务，便于服务聚合与端口复用
- 客户端在发起 RPC 调用时，需要依据"服务名 + 方法名 + IP + 端口"四元组精确定位目标方法
- 服务名作为 RPC 寻址的命名空间，与方法名配合实现方法级别的路由
