## Description
`gen_cmake_file_lists` 是 Protobuf 项目为支持多 C++ 构建系统（目前主要为 [[entities/Bazel|Bazel]] 和 [[entities/CMake|CMake]]）而设计的核心机制之一。其核心思想是：输入文件列表由 Bazel 生成，输出为可被其他构建系统导入的格式，目前唯一支持的格式为 CMake 风格的文件。该规则接受一个名为 `src_libs` 的字典参数，将 Bazel 构建目标映射到对应的 CMake 变量名，支持的源类型包括 [[concepts/cc_dist_library|cc_dist_library]] 规则、`proto_library` 规则、单个文件、`filegroup` 规则，以及来自 [[entities/rules_pkg|rules_pkg]] 的 `pkg_files`/`pkg_filegroup` 规则。生成过程通过调用相应的 aspect（如 `cc_file_list_aspect`、`proto_file_list_aspect`）从 Bazel 目标中提取源文件和头文件信息，最终输出可被手写 CMake 构建脚本 `include` 的 `.cmake` 文件，其中包含 `*_files`、`*_srcs`、`*_hdrs` 等变量声明，供 CMake 定义库目标时使用。`gen_cmake_file_lists` 内部通过"fragment generator"为不同类型源生成对应的 CMake 变量声明，大部分实现在规则间共享；当需要新增构建系统时，开发者需新增规则类型并定义对应的 fragment generator 以覆盖新的源类型。
