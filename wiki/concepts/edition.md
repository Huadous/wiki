## Description
Editions 机制由 Protobuf 团队提出，其核心思想是：一个 Edition 是 protoc 及其后端所理解的全部 feature 的默认值集合。Editions 按年份编号（例如 2023、2024），团队承诺每年发布一个 Edition。每个 Edition 仅改变 feature 的默认值，不会引入新的行为或破坏现有代码。这一设计直接借鉴自 [[concepts/life-of-an-edition|Rust 的 Edition 系统]]，Rust 将语言演进内置为核心设计，Protobuf 同样希望以此机制让生态平稳向前推进。

根据新来源 [[sources/editions-edition-lifetimes|editions-edition-lifetimes]]，当前 Editions 与 breaking changes 是相互分离的：所有版本相关的破坏性变更都通过 OSS 发布线的破坏性发布来承载。这一解耦带来一个重要的良好后果——行为保持兼容的 Edition 升级始终可以执行（相对于最新的 proto 语言而言）。换言之，只要 Editions 本身被精心设计为只调整默认值，用户从一个 Edition 升级到另一个 Edition 永远不会引入破坏性变更。该来源同时指出："Today, features and editions are largely disconnected from each other. We have a set of features that govern various behaviors, and they can all be used in any edition."——这表明 features 与 editions 之间的对应关系是松散的，同一组 features 可在任意 edition 中使用。

Editions 通过 [[concepts/features-option|features option]] 在文件级别及更低粒度上设置特性，并支持 [[concepts/feature-inheritance|Feature Inheritance]] 与 [[concepts/option-retention|Option Retention]] 等机制。每个文件的 Edition 由 [[concepts/edition-keyword|edition keyword]] 指定；若未指定，编译器默认按 proto2 处理。Editions 之间的顺序由 [[concepts/edition-total-order|Edition Total Order]] 保证，配合 [[concepts/minimum-required-edition|Minimum Required Edition]] 机制来约束加载行为。
