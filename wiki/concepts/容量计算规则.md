# 容量计算规则

## 定义
容量计算规则是 DynamicPartitionChannel 用于动态分配流量给不同分库方式的核心算法。该规则定义了不同类型 Channel 如何递归计算其容量值。

## 关键特征
- **递归计算**：容量的计算是递归进行的，从叶子节点（单个 server）向上聚合到根 Channel。
- **Channel 类型差异化计算**：不同类型的 Channel 采用不同的容量聚合策略：
  - 普通 Channel：容量等于所有 server 容量之和（单个 server 默认容量为 1）。
  - ParallelChannel / PartitionChannel：容量等于各 Sub Channel 容量的最小值。
  - SelectiveChannel：容量等于各 Sub Channel 容量之和。
  - DynamicPartitionChannel：容量等于其 Sub PartitionChannel 容量之和。
- **流量按比例分配**：多种分库方案可以共存，流量根据每种方案的总容量按比例分配，实现平滑迁移和扩缩容。
- **动态调整**：容量随底层 server 的增减自动变化，无需人工干预配置。

## 应用
- **多分库方案共存**：允许在同一个 DynamicPartitionChannel 中混合使用不同的分库策略（如哈希分库和范围分库），并根据容量比例分配流量。
- **平滑扩缩容**：新增或移除 server 时，容量自动调整，流量逐渐迁移，避免瞬间压力变化。
- **灰度发布与迁移**：通过调整特定分库方案的容量，控制其承接的流量比例，实现渐进式上线的灰度策略。
