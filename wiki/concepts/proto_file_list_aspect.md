## Description
`proto_file_list_aspect` 是 Protobuf 项目为支持多构建系统而设计的 Bazel aspect，运行时会遍历 `proto_library` 目标，同时提取 `proto_library` 自身的 `srcs` 与 Protobuf 编译器（protoc）针对每个 `.proto` 源文件所产出的预期文件路径，包括 `.proto.pb.cc`、`.proto.pb.h` 以及 `-descriptor-set.proto.bin` 等。它通过一个名为 `ProtoFileList` 的自定义 provider 将这些信息对外暴露，从而弥补了原生 `proto_library` 不直接给出 protoc 全部产物路径的不足。该 aspect 单独使用时用途有限，但可以被自定义规则实例化，使得普通的 `BUILD.bazel` 目标能够基于 aspect 收集到的信息产出对应的构建结果。在 Protobuf 的多构建系统支持架构中，`proto_file_list_aspect` 与 `cc_file_list_aspect` 协同工作，分别负责 proto 库和 C++ 源库两类构建目标的源文件信息提取，共同为 Bazel + CMake 等混合工作流提供完整的数据基础。
