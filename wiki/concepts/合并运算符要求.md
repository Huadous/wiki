# 合并运算符要求

## 定义
合并运算符要求是 [[entities/bvar::Reducer|bvar::Reducer]] 模板类对二元运算符所施加的强制性约束标准。该标准规定，用于将多个线程私有值合并为单一结果值的二元运算符 `Op` 必须同时满足结合律、交换律以及无副作用三项性质，以确保并行合并的结果与线程私有数据的分布方式无关。

## 关键特征
- **结合律（Associative）**：`a Op (b Op c) == (a Op b) Op c`，运算顺序不影响最终结果。
- **交换律（Commutative）**：`a Op b == b Op a`，参与运算的值的顺序不影响最终结果。
- **无副作用（No side effects）**：当 `a` 与 `b` 固定时，`a Op b` 的结果永远不变，运算符不修改任何外部状态。
- **违反后果**：若运算符不满足以上任意一条，合并结果为**未定义行为**（undefined result）。
- **典型反例**：减法不满足结合律，也不满足交换律，因此不能用作 Reducer 的合并运算符。
- **预定义符合要求的子类**：[[entities/bvar::Adder|bvar::Adder]]（运算符 `+`）、[[entities/bvar::Maxer|bvar::Maxer]]（运算符 [[entities/std::max|std::max]]）、[[entities/bvar::Miner|bvar::Miner]]（运算符 [[entities/std::min|std::min]]），三者分别以 `std::numeric_limits<T>::min()` / `max()` 作为幺元值。

## 应用
- 适用于 [[entities/bvar::Reducer|bvar::Reducer]] 模板类及其所有预定义子类的设计与使用场景。
- 用于多线程环境下对分散在各线程中的局部数据进行无锁/低竞争的合并统计，避免因线程调度或数据分布差异导致结果不确定。
- 作为 bvar 库自定义新型 Reducer 子类时遵循的接口契约，是正确实现并行聚合的基础规范。
- 与 [[concepts/衍生变量|衍生变量]] 机制配合使用，确保统计指标在并行环境下的语义一致性。
