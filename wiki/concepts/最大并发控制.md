# 最大并发控制

## 定义
最大并发控制是 brpc 提供的一种 server 级和 method 级的并发度限制机制。当 server 或 method 同时处理的请求数超过设定的并发度限制时，server 会立即向 client 回复 **brpc::ELIMIT** 错误，让 client 重试另一台 server，从而避免过载。最大并发度的计算遵循 Little's Law：最大并发度 = 极限 QPS × 低负载延时。

## 关键特征
- 支持 server 级和 method 级两层并发度控制
- 超过并发度限制时立即返回 `brpc::ELIMIT` 错误，不进入请求处理队列
- 通过快速失败促使 client 切换到其他 server，实现负载分散
- 配置方式灵活：可通过 `ServerOptions.max_concurrency`、`ServerOptions.method_max_concurrency` 或运行时调用 `server.MaxConcurrencyOf()` 进行设置
- 在传统的同步 server 中，最大并发不会超过工作线程数

## 应用
- 防止单个 server 因请求堆积导致响应延迟飙升或资源耗尽
- 在多 server 集群中，通过快速拒绝（fail-fast）将流量引导到负载较低的 server
- 配合 client 端的重试策略，构建具有弹性伸缩能力的服务集群
- 针对热点 method 设置独立的 method 级并发上限，避免某一方法挤占整个 server 的处理能力
