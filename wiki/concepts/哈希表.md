# 哈希表

## 定义
哈希表（Hash Table / HashMap）是一种通过哈希函数将键（key）映射到表中特定位置（桶 / bucket）以加速查找的数据结构。其基本原理是利用哈希函数把不同的 key 分散到不同的区间，查找时通过 key 的哈希值快速缩小查找范围。在使用恰当参数的前提下，哈希表在大部分时候能在 O(1) 时间内把一个 key 映射为对应的 value，但该"O(1)"在不同的实现中性能差异很大。哈希表的实现主要涉及两大核心问题：哈希函数设计与冲突解决（collision resolution）。

## 关键特征
- 通过哈希函数将 key 映射到桶区间，期望实现平均 O(1) 的查找、插入与删除复杂度。
- 实际性能高度依赖实现细节：哈希函数质量、冲突解决策略、负载因子、扩容策略等都会显著影响常数因子乃至最坏情况复杂度。
- 核心子问题分为两类：
  - **哈希函数设计**：要求分布均匀、计算迅速，常以 [[concepts/雪崩效应]] 作为衡量指标。
  - **冲突解决**：常见路径包括 [[concepts/开链哈希]]（separate chaining）、[[concepts/闭链哈希]]（open addressing）、混合方案以及多次哈希（multiple hashing）。
- [[concepts/负载因子]]（load factor）是关键调参指标：负载因子过高会显著放大冲突代价，过低则浪费空间。
- 在工业实践中，[[concepts/开链哈希]] 往往比"名声更快"的 [[concepts/闭链哈希]] 更稳定，原因在于开链实现对哈希质量与扩容过程的退化更不敏感。

## 应用
- 编程语言运行时与标准库的核心容器（如 C++ `std::unordered_map`、Java `HashMap`、Python `dict`、Go `map`）。
- 数据库索引、缓存系统、符号表、集合去重、关联数组等需要键值映射的场景。
- 高性能服务端组件中的热点结构，例如 brpc 中提供的 [[entities/flatmap|FlatMap]]（开链哈希）、[[entities/alignhashmap|AlignHashMap]]（闭链哈希）与 [[entities/cowhashmap|CowHashMap]] 等专用实现，分别面向不同的延迟、空间与并发需求。
- 分布式系统中的分片（sharding）/一致性哈希、负载均衡、特征查找与去重等。
