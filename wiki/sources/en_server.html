brpc Server 使用教程 - Summary

来源

  • Original file: en_server.md
  • Ingested: 2026-06-12

核心内容

本文档是 Apache brpc 框架的服务器端使用指南，详细说明了如何创建和运行 RPC 服务。内容涵盖：从定义 .proto 文件、实现 protobuf 服务接口、将服务添加到 brpc::Server、启动和停止服务器，到各种高级设置（如 SSL、限流、身份验证、协议支持等）。文档还解释了同步服务与异步服务的区别、RAII 技巧（如ClosureGuard）、以及多种数据局部存储机制。此外，提供了协议支持列表、常见问题解答和服务器端工作流程图，是开发基于 brpc 的服务器的权威参考。

关键实体

  • brpc：Apache 软件基金会下的高性能 RPC 框架，核心产品
  • brpc Server：框架中用于承载 RPC 服务的核心类
  • brpc::Controller：服务器端编程中用于传递元数据的核心接口
  • nginx：高性能反向代理服务器，可用于保护内置服务
  • curl：用于测试 HTTP+JSON 接口的命令行工具
  • hulu_pbrpc协议：默认启用的外部 RPC 协议
  • sofa_pbrpc协议：默认启用的外部 RPC 协议

关键概念

  • Protocol Buffers：定义服务接口和消息格式的标准
  • 同步服务与异步服务：两种服务实现模式
  • RAII与ClosureGuard：资源管理和回调执行机制
  • 限流：通过 max_concurrency 控制的并发保护机制
  • 数据局部存储：session-local、server-thread-local、bthread-local 三种机制
  • bthread：brpc 的轻量级协程实现
  • 协议自动检测：同一端口自动识别多种协议
  • TLS与Authenticator：安全与认证机制
  • Builtin Services：内置的诊断和管理 HTTP 接口
  • ServerOptions：服务器配置结构体
  • baidu_std协议：百度内部标准 RPC 协议
  • 2与gRPC：现代通信协议支持

要点

  • 在 .proto 文件中定义服务接口，通过 protoc 生成 C++ 代码
  • 实现服务接口时，必须正确处理 done->Run()，推荐使用 ClosureGuard 确保资源释放
  • 同步服务在 CallMethod 返回前完成处理，异步服务将 done 保存到后续事件
  • brpc Server 支持添加多个服务，且可以在同一端口上自动检测多种协议
  • 服务器限流（max_concurrency）可防止过载，提供服务器级和方法级两种设置
  • 提供三种局部存储机制：session-local、server-thread-local、bthread-local
  • 支持 SSL、身份验证、压缩、附件等功能
  • pthread 模式用于兼容不支持 bthread 的代码（如 JNI）
  • fork 无 exec 的情况需要特殊注意，建议在初始化所有 Server/Channel 前进行
  • 内置服务可通过 internal_port 隐藏，或通过配置关闭特定内置服务
