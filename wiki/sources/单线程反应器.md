## 别名
- single-threaded reactor
- Redis 单线程模型

## 要点
- 单线程反应器通过单线程事件循环和非阻塞IO实现高并发，但仅能利用一个CPU核心
- Redis 服务器是该架构的经典实现，brpc 性能测试验证了 CPU 瓶颈
- 当客户端线程数达到 50 时，Redis CPU 使用率接近 100%，QPS 显著下降
- hiredis 连接池模式性能低于单连接共享模式，因为单线程无法并行利用多个连接
- 该架构同样适用于 Nginx 早期版本和 Node.js 等系统
- 所有 IO 事件在一个线程中串行处理，避免了锁竞争的开销
- 在计算密集型任务中容易成为瓶颈，但在 IO 密集型场景中表现出色
- [[entities/en-http-service|en_http_service]] 与该模式无直接关联，但可作为 HTTP 服务实现的背景参考
