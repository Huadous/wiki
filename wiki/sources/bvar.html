核心内容

本文档介绍 brpc 项目内置的多线程计数器类库 bvar。bvar 位于 brpc 源代码树的 src/bvar 目录下，专门为多线程环境下的计数和监控场景设计。它利用 thread local存储 技术让每个线程只累加私有变量、读取时再合并所有线程数据，从而避免 cache bouncing 问题，将写竞争转移至读端，写入开销极低（约 20 纳秒）且几乎不随线程数增长。bvar 性能显著优于百度内部旧版计数器库 UbMonitor（300 次 bvar 累加的开销才抵得上一次动态 UbMonitor），同时也快于竞争频繁的 原子操作。文档涵盖其设计原理、单维度与多维度形式（mbvar）、HTTP 接口 /vars 的查看方式、文件导出（dump功能）以及与 prometheus、百度 noah 等监控系统的集成方法。

bvar 是 brpc 可观测性体系的核心组件，被 brpc 大量用于提供各类运行时统计数值。它是 brpc 的内置模块，因此也被称为 bvar计数器库，方便用户记录和查看程序中的各类数值。

关键实体

  • bvar — Apache brpc 内置的多线程计数器类库（别名：bvar计数器库）
  • brpc — 集成并大量使用 bvar 的 RPC 框架
  • UbMonitor — 百度内部旧版计数器库，bvar 的性能对比基线
  • prometheus — 通过 /brpc_metrics 端点采集 bvar 指标的开源监控系统
  • apache — 管理 brpc 项目的开源组织
  • noah — 百度内部基于 bvar dump 文件的监控系统
  • bthread — brpc 框架的用户态线程库，提供多个 bvar 统计指标
  • iobuf — brpc 框架的 I/O 缓冲区，提供多个 bvar 统计指标

关键概念

  • cacheline — 理解 bvar 性能原理所需的基础缓存概念
  • cache-bouncing — 多线程竞争同一缓存行导致的性能问题
  • thread-local存储 — bvar 避免 cache bouncing 的核心技术
  • 原子操作 — 性能对比中的一种基线方案
  • mbvar — 多维度版本的 bvar
  • dump功能 — 将 bvar 导出到文件供监控系统采集的机制
  • vars — brpc 提供 bvar 列表的 HTTP 端点，可通过 /vars 查看所有曝光的 bvar，或通过 /vars/VARNAME 查阅某个具体的 bvar
  • brpc_metrics — brpc 为 Prometheus 提供的指标抓取端点

要点

  • bvar 是 brpc 内置的多线程计数器类库，分为单维度 bvar 和多维度 mbvar 两种形式，是 brpc 可观测性体系的核心组件
  • bvar 采用 thread local 存储将写竞争转移至读端，写入开销极低（约 20 纳秒），适合写入频繁但读取低频的监控场景
  • 性能优势显著：相比 UbMonitor 几乎不增加程序开销，相比 原子操作 在高竞争下更快
  • brpc 集成了 bvar，可通过 HTTP 接口 vars（/vars 列出所有 bvar，/vars/VARNAME 查询单个 bvar）查看曝光的计数器
  • 支持三种数据消费方式：HTTP 接口 vars 网页查看、文件 dump功能 导出、与 prometheus 通过 brpc_metrics 端点集成
  • bvar 不能完全替代所有计数器，因为读操作需合并所有线程数据，比写更慢
